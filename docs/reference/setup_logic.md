# Setup Logic

Setup logic is the process of using:

- player count
- outsider count
- claims
- role interactions
- mechanical results

to infer what role distribution is likely in play.

## Why setup logic matters

In many games, the setup itself contains major clues about:

- whether a Baron is present
- whether a Drunk is likely
- how believable certain claims are
- what misinformation patterns are possible

## Key questions

- How many Outsiders should exist?
- Do claims match expected numbers?
- What roles would explain this structure?
- Is a hidden Drunk plausible?
- Does this world require fewer assumptions than alternatives?

## Common mistakes

- ignoring Outsider count
- treating all claims equally
- assuming a clean world is always true
- failing to revisit setup assumptions as new info appears
